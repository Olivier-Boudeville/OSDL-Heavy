User's Guide of OSDL, version 0.2

Overview

This user's guide is still to be written, however here are a few hints.

Installation

You can install OSDL with a few methods.

The hard way

One have to compile from sources all OSDL-0.2's pre-requesites.

Pre requesites

  • SDL, the famous Sam Lantinga's library used for the low-level platform independance
  • Ceylan, our own generic-purpose library


The strange way

Go boldy where no one has gone before, fasten your seat-belt, and use LOANI.

Some hints

Filenames

The library itself, whether downloaded as binary or rebuilt from sources, is named :

  • libOSDL.so.0.0.2, it is the full shared library filename, with sonname and al, and the installation process should have created links libOSDL.so.0 and libOSDL.so pointing directly towards libOSDL.so.0.0.2.
  • similarly, there should be a libOSDL-0.2.0.a for the static library, along with its exact copy libOSDL.a, and with libOSDL.la, which does respect GNU libtool's conventions (no links for them, .a and .la are different files)


Please keep in mind that OSDL is a multimedia-oriented library built on top of pre requesites, so you should install them priorly to have OSDL-0.2 working.
