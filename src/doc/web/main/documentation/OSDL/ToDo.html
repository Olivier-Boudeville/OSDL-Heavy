Technical to do list


  • generalize toString with level of detail.
  • integrate png support (thanks to SDL_Image) and integrate all useful third party libraries.
  • encourage OSDL being a finite state-machine, for example not to have to pass explicitly parameters such as clipping Rectangles.
  • make a review of which methods should be inlined, and use a profiling tool.
  • make and use Showable, Lockable and Collectable object in Ceylan, for example for Surface.
  • find or create a unit test framework, and use it
  • use a finite state-machine approach for at least OSDL::Video
  • organize modularization through a base class with module name, author, interface, and so on, which can enforce Singleton pattern if necessary (look at Ceylan)
  • look whether this modularization could not allow to link to shared libraries (ex : .so) at runtime, in order to discover and use new objects (new implementations of known interfaces, maybe new interfaces ?) Kernel modules may be of use, too.
  • provide LOANI, an automatic installer for OSDL, Ceylan and their pre requesites
  • integrate Ceylan's new Log System in OSDL
